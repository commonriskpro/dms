## 2. DB migration verification plan

1. **Which DB tests use:** At runtime, `lib/db.ts` uses `TEST_DATABASE_URL ?? DATABASE_URL` and appends `connection_limit=2` when `TEST_DATABASE_URL` is set. `jest.setup.ts` loads `../../.env.local` (repo root); it does **not** set `DATABASE_URL`. So the effective URL is from `process.env.TEST_DATABASE_URL` (if set in .env.local) or `process.env.DATABASE_URL`. For integration runs, set `TEST_DATABASE_URL` in `.env.local` to the test Postgres URL.
2. **Verify migrations applied:** From repo root, run against the **same** URL as tests:
   - `cd apps/dealer && dotenv -e ../../.env.local -- npx prisma migrate status` (uses DATABASE_URL from .env.local; jest.setup sets DATABASE_URL=TEST_DATABASE_URL when TEST_DATABASE_URL is set, so tests and this command use same DB when .env.local has TEST_DATABASE_URL). Ensure no pending migrations.
3. **Apply if needed:** `npm run db:migrate` uses `.env.local` and `DIRECT_DATABASE_URL` or `DATABASE_URL`. For the **test** DB: set `DATABASE_URL` (or `DIRECT_DATABASE_URL`) in `.env.local` to the same URL as `TEST_DATABASE_URL`, then run `npm run db:migrate`. Or temporarily `DATABASE_URL=<test-url> npm run db:migrate`.
4. **Document:** See DEALER_TESTING_GUIDE / runbooks: "For integration tests, apply dealer migrations to the DB pointed to by TEST_DATABASE_URL; use DATABASE_URL= that value and run db:migrate."

## 3. Prisma client generation / resolution verification plan

1. **Generation:** Prisma client is generated by `prisma generate` in `apps/dealer`; output is to repo root `node_modules/@prisma/client` (default when no output in schema). So from `apps/dealer`, resolution goes to `../../node_modules/@prisma/client`.
2. **When to generate:** Dealer has `pretest: "prisma generate"` so `npm run test` / `npm run test:dealer` runs Prisma generate before Jest. Also: `npm run build` (dealer), `npm run db:generate`, and `npm run prisma:generate:dealer` (root) generate the client.
3. **Jest resolution:** Jest runs with `cwd` = apps/dealer. `@prisma/client` resolves to root node_modules. Pretest ensures the client exists before any test file loads it, avoiding ModNotFoundError.
4. **Safeguard in place:** `pretest` in apps/dealer runs `prisma generate` before every test run.
